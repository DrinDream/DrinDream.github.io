2. Recommendation

Based on your requirements for high-quality paper analysis, multi-model support (Claude, Gemini, DeepSeek, ChatGPT), and innovation via debate, the best open-source project for this specific workflow is “The AI Scientist” by Sakana AI.

While primarily designed to conduct research, its modular architecture (specifically the Ideation and Peer Review modules) perfectly matches your need to generate and critique innovation ideas.

Here is the detailed recommendation and how to configure it for your goal:

1). Top Recommendation: The AI Scientist (by Sakana AI)

This project is currently the state-of-the-art open-source framework for automating scientific discovery. It explicitly uses a multi-agent loop to generate ideas and then “review” (debate/critique) them.

  • GitHub: SakanaAI/AI-Scientist

  • Why it fits your need:

    • Multi-Model Backend: It natively supports DeepSeek (very cost-effective for the heavy reasoning/debate loops), Claude Sonnet 3.5, GPT-4o, and Gemini.

    • Innovation Engine: It has a dedicated “Idea Generation” module that takes existing papers (seeds) and brainstorms novel directions.

    • Debate/Critique: It features a “Reviewer Agent” that acts as a NeurIPS/ICLR reviewer to harshly critique the generated ideas and papers, effectively simulating a debate to filter for “practical innovation.”

How to use it for your specific workflow:

You don’t need to run the full code-generation loop. You can isolate the Idea Generation and Review phases:

  1. Input: Place your high-quality papers (PDFs/text) as “seed ideas” or “templates.”
  2. Configuration: Edit config.yaml to assign different models to different roles.
    • Idea Generator: Use Claude 3.5 Sonnet or DeepSeek-V3 (high creativity).
    • Reviewer/Debater: Use GPT-4o or Gemini 1.5 Pro (strong logic/critique).
  3. Execution: Run the generate_ideas.py script. The agents will:
    • Read the context.
    • Propose new research directions.
    • Search Semantic Scholar to ensure novelty.
    • Critique the feasibility (the “debate” aspect).

2). Best Framework for Custom Debate: MetaGPT

If you want a pure “Round Table Debate” where Agent A (Claude) argues with Agent B (DeepSeek) about a specific paper, MetaGPT is the most flexible framework to implement this without the heavy overhead of Sakana’s full scientific pipeline.

  • GitHub: geekan/MetaGPT

  • Key Feature: It has a built-in Debate example and a Researcher role.

  • How to implement:

    • Role 1 (The Analyst): Create a “Paper Reader” agent (using Gemini 1.5 Pro for its large context window) to ingest the PDF and summarize key points.

    • Role 2 (The Innovator): Create a “Proponent” agent (using DeepSeek) to propose an application based on the summary.

    • Role 3 (The Critic): Create an “Opponent” agent (using GPT-4) to find flaws in the proposal.

    • The Workflow: Use the Debate class in MetaGPT to let Role 2 and Role 3 argue for $N$ rounds, refining the idea into a practical solution.

3). Best for Deep Analysis & Synthesis: Stanford STORM

If your priority is less on “generating new ideas” and more on “deeply understanding and connecting” the papers first, STORM is superior.

  • GitHub: stanford-oval/storm
  • Key Function: It builds a “Wikipedia-style” deep dive by simulating a conversation between a Writer and an Expert.
  • Innovation Angle: Use Co-STORM (Collaborative STORM). You can upload your papers, and the “Expert” agent will debate you (or another agent) about the content, forcing you to clarify and refine the concepts.
